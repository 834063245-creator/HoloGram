# 横向对比报告：开源代码依赖可视化工具

**视角：重度用户 | 日期：2025-06-12 | 对标对象：HoloGram (Tauri 2 + Python + Three.js 3D 星图)**

---

## 〇、HoloGram 实际功能清单（先搞清楚自己有什么）

| 功能 | 实现方式 |
|------|---------|
| **多语言解析** | Tree-sitter 15 语言（Python / JS / TS / TSX / Go / Rust / Java / C / C++ / Ruby / C# / Kotlin / Swift / PHP / Lua），自动按扩展名分发 |
| **3D 星图渲染** | Three.js + UnrealBloomPass 泛光 + `sqrt(n)` 球壳缩放，节点恒占视觉空间 |
| **三视觉模式** | minimal / standard / full，各自独立实例，切换即重建 |
| **搜索** | 工具栏搜索框 → `focusNode(query)`，匹配后镜头飞过去+脉冲高亮 |
| **节点交互** | 点击→详情卡片（名称/类型/耦合度/文件路径）+ 四个按钮：**[打开]** → FileViewer、**[问 Agent]** → Chat 面板、**[波及]** → 影响范围可视化、**[聚焦]** → 镜头贴近 |
| **波及模式 (Blast)** | 选中节点后计算依赖距离，按距离分层着色，展示"改这个会炸多远" |
| **宇宙折叠 (Fold)** | 星团→星系两级视图，进入/退出星系，大项目不乱 |
| **浮动代码窗口** | FileViewer：Monaco 编辑器 + 多标签页 + 拖拽/缩放 + Ctrl+S 保存 + diff 视图 + 语法高亮 |
| **文件树↔星图联动** | 左侧文件树点击 → 3D 图中对应节点高亮 + 其余变暗 |
| **Agent↔星图联动** | Chat 里 Agent 说完 → 自动高亮相关节点，镜头飞过去 |
| **Agent 代码分析** | 点节点→"问 Agent"→自动生成分析 prompt（节点名/类型/度/位置），Agent 回答依赖关系和修改影响 |
| **消息传输** | MessagePack 二进制压缩，大项目加载快 10× |
| **桌面原生** | Tauri 2 壳，比 Electron/Java 轻，启动快，内存省 |
| **零配置** | 打开项目目录 → 自动解析 → 出图，不需要手动准备任何文件 |

## 二、逐项目深度对比

### 1. dep-tree — 最接近 HoloGram 的同赛道对手 🔥

**一句话：** "代码库越脏，图越乱"——用 3D 力导向图暴露出模块耦合问题。

| 维度 | dep-tree | HoloGram |
|------|----------|----------|
| **交互** | 3D 旋转/缩放，节点拖拽，悬停高亮。有终端 TUI 模式。无节点折叠/展开。 | 3D 旋转/缩放/平移，悬停高亮邻居边。点击弹详情卡片。**有波及模式和宇宙折叠**——dep-tree 没有这两种交互。 |
| **可操作** | 点击看文件路径和依赖列表。**能进 CI 做架构规则校验**——独门绝活。不能跳编辑器。 | 点击→详情卡→**[打开]** 浮动 Monaco 编辑器、**[问 Agent]** 分析、**[波及]** 影响范围、**[聚焦]** 镜头飞入。**能看源码能改源码能问 AI**，但不能进 CI。 |
| **视觉** | Three.js 3D，默认配色干净。干净项目=清晰团簇，脏项目=毛线球。无动画过渡。 | Three.js 3D + UnrealBloomPass 泛光 + 球壳缩放 + 宇宙隐喻。三种模式切换。视觉上**碾压**。 |
| **性能** | 力导向布局大项目卡，数百节点可用，上千吃力。 | MessagePack 压缩传输，加载快。但 3D 渲染上限差不多（都是 Three.js）。 |
| **痛点** | 社区小，文档少。只到模块级。配置文件学习曲线陡。 | 需要两步操作才能看代码（vs SourceTrail 点一下）。还没有 CI 集成。 |
| **语言** | 5 种（TS/JS/Python/Rust/Go） | **15 种** |
| **胜负** | **CI 集成**是 dep-tree 唯一强过 HoloGram 的地方。其余所有维度 HoloGram 都持平或领先。 |


### 2. Emerge — 浏览器里的多语言分析仪

**一句话：** 打开浏览器就能分析 12 种语言的代码库，D3 力模拟 + Canvas 渲染。

| 维度 | Emerge | HoloGram |
|------|--------|----------|
| **交互** | D3 力模拟 + Canvas。缩放/平移流畅。Louvain 社区检测+凹包轮廓。支持搜索过滤。 | 3D 沉浸感更强。有波及模式和宇宙折叠。但**没有语义搜索和聚类轮廓**。 |
| **可操作** | 悬停看 SLOC/扇出风险分。语义搜索（TF-IDF）。导出 GraphML/JSON/D3。不能跳编辑器。 | 详情卡→FileViewer→读源码/改源码/问 Agent。操作深度**远胜** Emerge。 |
| **视觉** | 2D Canvas + 热力图配色。凹包轮廓让边界清晰。信息密度高。 | 3D 星图，视觉上不在一个量级。 |
| **性能** | Canvas 比 DOM 快，几千节点可行。但浏览器吃内存。 | 桌面原生，不受浏览器内存限制。 |
| **痛点** | 初次分析大项目慢。配置项多且散。 | 聚类可视化是 Emerge 的优势点，但语义搜索在依赖图场景用处有限。 |
| **语言** | 12 种 | **15 种** |
| **胜负** | Emerge 的**Louvain 聚类+凹包轮廓**值得借鉴。其余维度 HoloGram 领先。语义搜索对依赖图不是刚需。 |


### 3. Gephi — "图可视化的 Photoshop" 🎨

**一句话：** 学术界和数据分析师的最强图工具，但程序员没法直接用。

| 维度 | Gephi | HoloGram |
|------|-------|----------|
| **交互** | OpenGL 加速，实时拖拽/缩放/过滤/重着色/重布局。**过滤系统极强**（属性/拓扑/范围）。交互自由度全场最高。 | 3D 旋转/缩放+波及+折叠。过滤系统**没有**——大项目所有节点一锅粥。 |
| **可操作** | 一键计算介数中心性、模块度、连通分量等图指标。按度/社区着色。导出 PNG/PDF/SVG。**不能导出交互式 Web**（最大槽点）。 | FileViewer + Agent 分析，但**没有图指标计算**。零配置从代码到图是核心优势。 |
| **视觉** | 布局算法最多：ForceAtlas2、Fruchterman-Reingold、Yifan Hu。可发表级别。 | 3D 星图独一份，但布局只有力导向。Gephi 的布局多样性是碾压的。 |
| **性能** | 5 万~30 万节点流畅。崩溃是老毛病（Java）。 | 节点规模不如 Gephi（Three.js vs OpenGL），但启动快不崩。 |
| **痛点** | 学习曲线陡。文档烂。**不是程序员工具**——你得自己把代码库转成节点/边 CSV。 | 过滤系统和图指标是 Gephi 的绝对压制区。 |
| **胜负** | Gephi 在**纯可视化自由度**和**图分析深度**上碾压所有对手。但它是通用工具，不是代码工具——HoloGram 的零配置代码到图是 Gephi 永远做不到的。两者其实是互补关系。 |


### 4. SourceTrail — 已死的传奇 👻

**一句话：** 如果它还活着，会是这个品类的绝对标杆。可惜 2021 年归档了。

| 维度 | SourceTrail | HoloGram |
|------|-------------|----------|
| **交互** | 左侧图视图 + 右侧代码视图，**永久并排**。点节点→代码跳转。模糊搜索极强（驼峰/部分匹配/正则）。 | 点击节点→详情卡→点"打开"→FileViewer 浮动窗口。**两步操作 vs SourceTrail 一步**，但窗口可拖拽/多标签/保存。搜索是精确匹配，不如 SourceTrail 的模糊搜索好用。 |
| **可操作** | 展开/折叠子图。与 VS Code/IntelliJ/Vim/Sublime 联动。 | FileViewer 可读可改可保存。Agent 可分析。波及模式。宇宙折叠。**Agent 集成**是 SourceTrail 完全没有的能力。 |
| **视觉** | 2D 平面图，按类型着色（类=蓝/函数=黄/变量=绿）。功能性极强，不炫。 | 3D 星图碾压，但 SourceTrail 的功能性配色值得借鉴。 |
| **性能** | 唯一能在 Firefox 代码库上跑的工具。 | MessagePack 传输快，大项目加载有优势。 |
| **痛点** | **死了。** 新语言特性不支持。编译越来越难。 | 搜索不如 SourceTrail 模糊。并排视图不如 SourceTrail 一步直达。 |
| **胜负** | SourceTrail 的**图-代码并排联动**和**模糊搜索**是这个品类的 UX 教科书。HoloGram 的浮动窗口方案是可行的替代——功能上没少，但少了一步直达的快感。Agent 集成是 HoloGram 对 SourceTrail 的降维打击。 |


### 5. dependency-cruiser — JS/TS 架构警察 🚔

**一句话：** 这不是可视化工具，是架构规则的 linter——附带了一个静态图。

| 维度 | dependency-cruiser | HoloGram |
|------|-------------------|----------|
| **交互** | 零。生成的 SVG/HTML 是只读的，仅 hover 高亮路径。 | 3D 可旋转/缩放/点击/波及/折叠。交互维度碾压。 |
| **可操作** | 核心是规则引擎：`cannot-reach`/`must-reach`/循环依赖检测。CI 原生集成。 | FileViewer + Agent + 波及。图探索碾压，但**没有架构规则**。 |
| **视觉** | Graphviz 渲染。功能导向，不追求美观。 | 碾压。 |
| **语言** | JS/TS/TSX/Vue/Svelte 等前端栈 | **15 种** |
| **胜负** | 完全不同赛道。它是**规则引擎**，HoloGram 是**探索工具**。互补：cruiser 在 CI 卡规则，HoloGram 让人理解结构。 |


### 6. pydeps — Python 模块依赖的"够用"方案 🐍

**一句话：** `pip install pydeps && pydeps myproject`，两分钟出 SVG。然后你就不想再用它了。

| 维度 | pydeps | HoloGram |
|------|--------|----------|
| **交互** | SVG hover 高亮。有且仅有这个交互。 | 碾压。 |
| **可操作** | 循环依赖检测（Kosaraju SCC）。聚类外部依赖。过滤噪声。导出 JSON/DOT/SVG/PNG。 | 碾压。 |
| **视觉** | Graphviz 渲染，实用但不美。 | 碾压。 |
| **性能** | 默认只展开 2 层（`--max-bacon=2`），很聪明。 | 大项目 Analysis 阶段慢于 pydeps 的 AST 扫描。 |
| **语言** | 仅 Python | **15 种** |
| **胜负** | 不是一个重量级。pydeps 是 CLI 快照工具，HoloGram 是交互式桌面应用。pydeps 唯一优势是**快**——两分钟出结果。 |


### 7. code2flow — 函数调用链的快照相机 📸

**一句话：** 用 AST 生成函数调用图，输出静态图片。快，但浅。

| 维度 | code2flow | HoloGram |
|------|-----------|----------|
| **交互** | 零。输出 PNG/SVG 静态图片。 | 碾压。 |
| **可操作** | 控制深度、分组合并。找孤立函数和循环依赖。 | 碾压。 |
| **视觉** | Graphviz 渲染，不可微调。 | 碾压。 |
| **性能** | AST 解析极快。 | Analysis 阶段慢。 |
| **语言** | Python/JS/TS/Go/Ruby/PHP/C++ | **15 种**（多一倍） |
| **胜负** | code2flow 是"拍一张 X 光片"，HoloGram 是"走进 3D 模型里逛"。目的完全不同。 |

### 评分依据

**交互质感 (5):** HoloGram 的 3D 旋转/缩放/悬停高亮/波及分层/宇宙折叠/镜头飞行动画，交互丰富度仅次于 Gephi 的过滤系统。两次点击才能看代码（vs SourceTrail 一次）是扣分项。

**视觉质量 (5):** 3D 星图隐喻 + UnrealBloomPass 泛光 + 球壳缩放 + 深空背景，这个品类里没有任何工具在视觉上能接近。Gephi 强在可发表级的学术图，HoloGram 强在沉浸式空间体验——赛道不同，平分秋色。

**性能 (3):** Three.js 渲染几千节点是上限。比不上 Gephi 的 OpenGL 30 万节点。MessagePack 压缩传输是加分项，但渲染瓶颈在 WebGL/Three.js。

**功能深度 (4):** 15 语言解析 + FileViewer + Agent 集成 + 波及模式 + 宇宙折叠 + 文件树联动 + git diff。缺的是：图指标计算（介数中心性等）、过滤系统、语义搜索。Agent 集成是这张表里**唯一的 AI 能力**，拉回一分。

**上手体验 (4):** 零配置→打开项目→出图。浮动窗口、详情卡片都直觉可用。扣分项是搜索只做精确匹配（没有 SourceTrail 的模糊搜索）和两次点击才能看代码（vs 并排视图的一步到位）。

## 四、关键洞察

### HoloGram 的独特优势（别人做不到的）

1. **15 语言零配置解析** — 覆盖度仅次 SourceTrail（后者靠插件体系），dep-tree 5 种 / Emerge 12 种。打开目录就出图，Gephi 需要手动准备 CSV
2. **3D 星图隐喻** — 全场唯一用宇宙/星图做代码依赖可视化的。Gephi/dep-tree 是"图"，HoloGram 是"星系"
3. **Agent 深度集成** — 点击节点→问 Agent→Agent 回答完自动高亮相关节点并镜头飞过去。**全场唯一的 AI 能力**，SourceTrail 做不到，dep-tree 做不到
4. **波及模式** — 从选中节点计算依赖距离，按距离分层着色，直观回答"改这个会炸多远"
5. **宇宙折叠** — 大项目星团→星系两级视图，进入/退出，不乱
6. **球壳缩放 + MessagePack** — `sqrt(n)` 节点空间恒定 + 二进制传输快 10×

### HoloGram 应该从别人那里偷的功能

| 来源 | 功能 | 优先级 | 为什么 |
|------|------|:---:|------|
| **SourceTrail** | 模糊搜索（驼峰/部分匹配/正则） | 🔴 高 | 当前精确匹配在大项目里不好用 |
| **Gephi** | 过滤系统（按度/类型/社区筛选） | 🔴 高 | 大项目节点爆炸，没有过滤没法用 |
| **Gephi** | 图指标计算（介数中心性/模块度/连通分量） | 🟡 中 | 给节点"按重要性着色"会让星图信息密度翻倍 |
| **Emerge** | Louvain 聚类 + 凹包轮廓 | 🟡 中 | 和现有宇宙折叠互补，自动发现社区边界 |
| **dep-tree** | CI 集成 / 架构规则校验 | 🟢 低 | 让 HoloGram 从探索工具变成规则执行者 |
| **dependency-cruiser** | 依赖规则 DSL | 🟢 低 | 同上，CI 场景 |
| ~~Emerge 语义搜索~~ | ~~TF-IDF 关键词~~ | ❌ 不需要 | 依赖图里搜的是精确标识符（文件名/函数名），语义搜索是代码探索场景，不是依赖图场景 |

### HoloGram 的实际短板（诚实版）

1. **搜索不够智能** — 精确匹配。在 500+ 节点的项目里，记不住完整文件名就搜不到
2. **节点爆炸无过滤** — 没有按类型、度、社区筛选的过滤器。大项目进来所有节点一锅粥，只能靠宇宙折叠缓解
3. **两次点击才能看代码** — 点节点→详情卡→点"打开"。SourceTrail 是点一下就到。这不是功能缺失，是 UX 摩擦
4. **没有图指标** — 不能按介数中心性/扇出风险/模块度着色，少了"数据驱动"的分析维度
5. **性能上限** — Three.js vs Gephi OpenGL，注定节点规模差一个量级

## 五、结论

同赛道没有统治者。

SourceTrail 死于 2021，留下一个 UX 教科书式的双面板遗产。Gephi 是可视化王者但不是程序员工具。dep-tree 太小众且功能浅。Emerge 浏览器方案天生受限。

HoloGram 现在的综合实力（3D 星图 + 15 语言 + FileViewer + Agent + 波及 + 折叠）已经进入品类第一梯队。三个短板补齐——模糊搜索、过滤系统、图指标着色——就能从第一梯队冲到品类标杆。

**Agent 本身不是壁垒——API 调用谁都会。** 壁垒是 Agent 和 3D 星图的耦合深度：Agent 输出自动高亮节点 + 镜头飞到现场，不是"左边聊天框右边图"各玩各的。这种装配精度和持续打磨才是别人抄不走的。SourceTrail 当年靠双面板联动封神，也是同样的道理——不是某个零件厉害，是零件之间的咬合精准。
